# プロジェクト概要書（お客さん相談結果）

## プロジェクト名

勤怠管理システム（デモ）

## 背景・目的

SIer のシステム開発において、AI 駆動開発（AI-DLC ベース）と TDD を組み合わせた開発プロセスを実践するデモプロジェクト。
「モダンな Java 開発はこう進める」を見せる題材として、勤怠管理を選定した。

### デモで示したいこと

- Spring Boot 3.x で開発しつつ、4.x 移行が最小限で済む実装方針
- AI 駆動 + TDD による SIer 水準の品質担保（テスト＝仕様書）
- ライト DDD によるドメイン分割
- フロントエンド（Next.js）との連携を含むフルスタック構成

## システム概要

従業員の日々の出退勤を記録・管理する Web アプリケーション。

### 想定ユーザー

| ロール | 概要 |
|--------|------|
| 一般社員 | 自分の出勤・退勤を打刻する。自分の勤怠履歴を確認する |
| 管理者 | 部署のメンバーの勤怠を確認・修正する。月次の勤怠集計を行う |

### 主な業務フロー（ヒアリングベース）

1. **出勤打刻** — 社員が出勤時に打刻する
2. **退勤打刻** — 社員が退勤時に打刻する
3. **勤怠履歴確認** — 自分の打刻履歴を日別・月別で確認する
4. **勤怠修正** — 打刻忘れや誤りを修正する（承認フロー付き）
5. **月次集計** — 管理者が月単位で勤怠データを集計・確認する

### スコープ外（今回やらないこと）

- シフト管理・スケジュール管理
- 給与計算連携
- 有給休暇の残日数管理
- モバイルアプリ（Web ブラウザのみ）
- 外部サービス連携（Slack 通知、カレンダー同期等）

## 技術構成（決定済み）

| レイヤー | 技術 | 備考 |
|---------|------|------|
| バックエンド | Spring Boot 3.x (Java 21) | 4.x 互換の書き方を徹底 |
| フロントエンド | Next.js (TypeScript) | App Router, Tailwind CSS |
| DB | PostgreSQL 17 | Flyway でマイグレーション管理 |
| インフラ | AWS CDK (TypeScript) | dev / prod の 2 環境 |
| 認証 | Spring Security | セッションベース or JWT（設計時に決める） |
| API 仕様 | SpringDoc OpenAPI (Swagger UI) | |

## 非機能要件（デモ水準）

| 項目 | 方針 |
|------|------|
| 可用性 | シングル構成（デモのため冗長化しない） |
| 性能 | 同時 10 ユーザー程度を想定。チューニングは行わない |
| セキュリティ | OWASP Top 10 対策は実装する。本番同等のセキュリティ設計 |
| テスト | カバレッジ 80% 以上。ArchUnit でレイヤー違反検出 |
| ログ | SLF4J + Logback。構造化ログは必要に応じて |
| 監視 | デモでは対象外（インフラ側で CloudWatch 程度） |

## 開発方針（合意済み）

| 項目 | 内容 |
|------|------|
| 開発プロセス | AI-DLC ベース + TDD |
| ドメインモデリング | ライト DDD（Entity, VO, Repository, Service） |
| アーキテクチャ | ドメイン分割レイヤード |
| テスト戦略 | テストファースト。テストが仕様書を兼ねる |
| ドキュメント | docs/path/ に過程記録、docs/ 以下に設計資料 |
