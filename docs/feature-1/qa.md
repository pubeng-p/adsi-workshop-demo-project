# 打刻メモ機能 — Q&A

## Q1: メモは出勤打刻時のみか？退勤打刻時にも入力できるか？

**A:** 1つのメモ欄（出退勤共通）。レコードに1つのメモフィールドがあり、出勤時にも退勤時にも入力・上書きできる。

## Q2: メモの入力は必須か？

**A:** 任意（空でもOK）。メモを書かなくても打刻できる。

## Q3: メモの最大文字数は？

**A:** 200文字。

## Q4: 打刻後にメモを編集できるか？

**A:** いつでも編集可能。ただし編集履歴を残す。

## Q5: 「編集履歴を残す」とはどの程度か？

**A:** 編集履歴テーブルを作成する。別テーブルに「いつ」「誰が」「何から何に」変更したか全履歴を保存する。

## Q6: 履歴画面でのメモの表示方法は？

**A:** テーブルにメモ列を追加して一覧表示する。

## Q7: メモの入力UIはどのような形か？

**A:** 打刻ボタンの上にテキストエリアを配置する。常に表示されており、入力してから打刻する形。

## Q8: ダッシュボードの打刻エリアにも同じメモ欄を表示するか？

**A:** 両方に表示する（打刻画面・ダッシュボード両方）。

## Q9: 編集履歴テーブルに「誰が」編集したかも記録するか？

**A:** 記録する。edited_by として編集した社員IDを保存する（将来管理者が部下のメモを編集するケースに対応）。

## Q10: メモの編集はどこから行うか？

**A:** 両方。履歴画面からも、今日の記録（TodayRecords）からも編集可能にする。
